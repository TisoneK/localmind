# Story 1 ŌĆö File Reading MVP (v0.1)

## Summary
As a user, I can upload a PDF (or DOCX/TXT) in the Web UI, ask a question about it, and receive an accurate answer sourced from the document ŌĆö all running locally with no internet.

## Acceptance Criteria

- [ ] User opens `http://localhost:8000` in a browser
- [ ] UI shows a file upload input and a chat input
- [ ] User uploads `report.pdf` and types "What are the key recommendations?"
- [ ] Model responds with content sourced from the PDF (not hallucinated)
- [ ] Entire interaction requires no internet connection
- [ ] Response streams token by token (not all at once)
- [ ] Session persists if the page is refreshed (history from SQLite)

## Out of Scope for this Story
- Web search
- Memory across sessions
- Code execution
- File writing
- CLI
- Model auto-detection
- Multi-file upload

## Tasks

### Task 1 ŌĆö Ollama adapter
**File:** `adapters/ollama.py`
- Wrap `POST /v1/chat/completions` with streaming
- Handle `ConnectError` (Ollama not running) with clear message
- Handle timeout
- Return `AsyncIterator[StreamChunk]`

**Done when:** `pytest tests/adapters/test_ollama.py` passes (mocked HTTP)

---

### Task 2 ŌĆö File reader tool
**File:** `tools/file_reader.py`
- Accept PDF, DOCX, TXT, CSV, plain-text code files
- Extract text using PyMuPDF (PDF), python-docx (DOCX), plain decode (rest)
- Chunk by token count with overlap (tiktoken cl100k_base)
- Return `FileAttachment` with populated `chunks` list
- Raise `ValueError` for files over `LOCALMIND_MAX_FILE_SIZE_MB`

**Done when:** `pytest tests/tools/test_file_reader.py` passes

---

### Task 3 ŌĆö Context builder
**File:** `core/context_builder.py`
- Accept `EngineContext` and `model_context_window`
- Assemble: system prompt + file chunks + conversation history
- Trim oldest history first when over token budget
- Never exceed `model_context_window - LOCALMIND_RESPONSE_RESERVE_TOKENS`

**Done when:** `pytest tests/core/test_context_builder.py` passes

---

### Task 4 ŌĆö Core engine
**File:** `core/engine.py`
- Load session history from `SessionStore`
- Classify intent via `intent_router.classify()`
- Parse file attachment via `tools.file_reader.parse_file()`
- Build context via `context_builder.build()`
- Stream response via `OllamaAdapter.chat()`
- Persist exchange to `SessionStore`

**Done when:** Engine can be instantiated and `process()` called in a test with mocked adapter

---

### Task 5 ŌĆö FastAPI routes
**Files:** `api/app.py`, `api/routes/chat.py`, `api/routes/health.py`
- `POST /api/chat` ŌĆö multipart form, returns SSE stream
- `GET /api/health` ŌĆö returns Ollama reachability + model name
- CORS allows localhost:5173 (Vite dev) and localhost:8000

**Done when:** `pytest tests/api/test_chat.py` passes with mocked engine

---

### Task 6 ŌĆö React UI (minimal)
**Files:** `ui/src/`
- Single page: file upload area + chat input + streaming message list
- Sends `POST /api/chat` as multipart form
- Reads SSE stream and appends tokens to the last message
- Shows "Uploading..." and "Thinking..." states
- No routing, no auth, no settings page (v0.1)

**Done when:** Manual test passes: upload a PDF, ask a question, see streamed response

---

### Task 7 ŌĆö SQLite session store
**File:** `storage/db.py`
- Schema already defined ŌĆö verify `PRAGMA foreign_keys` and WAL mode work
- `append()`, `get_history()`, `list_sessions()`, `delete_session()`

**Done when:** `pytest tests/storage/test_db.py` passes

---

### Task 8 ŌĆö README + install verification
**File:** `README.md`
- `pip install localmind` (or `pip install -e .`)
- `localmind start`
- Prerequisites: Python 3.10+, Ollama, at least one model pulled

**Done when:** A fresh install from the README instructions works end-to-end

---

## Definition of Done (Story level)
All 8 tasks complete AND the acceptance criteria above pass in a manual end-to-end test with a real PDF file and a running Ollama instance.

## Notes
- Do not add web search, memory, or code execution in this story even if it seems easy ŌĆö scope creep here delays v0.1 shipping
- The React UI should be as minimal as possible ŌĆö no design system, no component library in v0.1
- Streaming via SSE is mandatory ŌĆö polling is not acceptable
